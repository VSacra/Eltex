Перед началом работы скачайте файлы hello.c и Makefile с репозитория. 
Для дальнейшей работы будет необходимо прописать папку, в которую вы поместили файлы в переменную PATH. 
Для этого нужно найти системный файл ~/.bashrc и дописать в конец export PATH=$PATH:/путь/к/папке и применить изменения (пример на скринах 1-2).
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/1.png)
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/2.png)
После этого нужно зайти в папку со скаченными файлами и скомпилировать модуль ядра, прописав make all. 
Не забудьте, что для работы вам необходимы заголовки вашей версии драйвера в папке modules/lib/номердрайвера, так как makefile использует для компиляции makefile данного модуля ядра 
и просто дополняет модуль скомпилированным объектным файлом.  
После компиляции в папке с hello.c вы увидите несколько новый файлов, как например hello.ko. Пример на скрине 3. 
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/3.png)
Можно также посмотреть информацию о модулях, используя lsmod. (Скрин 4)
Чтобы вручную загрузить модуль необходимо прописать insmod hello.ko
Далее можно проверить, что модуль был смонтирован, захватив последнее сообщение в системном журнале, используя dmesg (скрин 4)
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/4.png)
Извлечь модуль: rmmod hello.ko
Также можно посмотреть сообщение, что ядро было извлечено (скрин 5)
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/5.png)
Посмотреть информацию об ядре можно с помощью modinfo hello.ko (скрин 6)
![](https://github.com/VSacra/Eltex/blob/main/module5/lab1/6.png)
